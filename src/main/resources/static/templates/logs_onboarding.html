FileBeat

Filebeat is a Lightweight log data shipper for local files/logs. Installed as an agent on your servers, Filebeat monitors the log directories or specific log files, tails the files, and forwards them either to Elasticsearch or Logstash for indexing. Being a lightweight log shipper agent, Filebeat is reliabale and utilizes fraction of the resources and can be easily installed on vast Operating Systems.

Instructions for Windows

Step 1: Download FileBeat
Step 2:Extract to any path. (Eg: C:\ProgramData\Filebeat ) this will be your filebeat homepath.
Step 3:Download the certificate 
https://raw.githubusercontent.com/tmcl/public-certificates/master/COMODORSADomainValidationSecureServerCA.crt
Step 4:configuration of Filebeat:
windows place it under 
C:\Program Files\Filebeat\filebeat.yml
Use configuration wizard for configuring your filebeat.yml
Step 5:Example Filebeat file: 
############################# Filebeat #####################################
filebeat:
  prospectors:
    -
      paths:
        - /path/to/log/file.log
      fields:
        codec: plain
        token: sgfdgsfdssfdshdjghsfdg
        type: json
      fields_under_root: true
      encoding: utf-8
      ignore_older: 3h
    -
      paths:
        - /path/to/json/file.json
      fields:
        codec: json
        token: sgfdghgfdhdfghgWGGGRfsdf
        type: type
      fields_under_root: true
      encoding: utf-8
      ignore_older: 3h
      
  registry_file: /var/lib/filebeat/registry
############################# Output ##########################################
output:
  logstash:
    hosts: ["listener.logz.io:5015"]
    
#  The below configuration is used for Filebeat 1.3 or lower
    tls:
      certificate_authorities: ['/etc/pki/tls/certs/COMODORSADomainValidationSecureServerCA.crt']
      
#  The below configuration is used for Filebeat 5.0 or higher      
    ssl:
      certificate_authorities: ['/etc/pki/tls/certs/COMODORSADomainValidationSecureServerCA.crt']

Logstash is an open source data collection engine with real-time pipelining capabilities. Logstash can dynamically unify data from disparate sources and normalize the data into destinations of your choice.

Instructions for Windows:
  • Download Logstash.
  • Configure logstash to add a token. Add the following section at the bottom of the filter section 
    mutate {
    add_field => { "token" => "fYuHpjTPJQHiyZrkeoIumQMDxYSyGsln" }
}
  • Configure logstash output to ship logs to TMCL Add the following section inside your output block: 
    tcp {
    host => "listener.TMCL"
    port => 5050
    codec => json_lines
}

Logstash configuration file should be saved with .conf extension Ex: sampleLog.conf
Then, run logstash and specify the configuration file with the -f flag. 


logstash -f simpleLog.conf

Syslog

Syslog stands for System Logging Protocol and is a standard protocol used to send system log or event messages to a specific server, called a syslog server. It is primarily used to collect various device logs from several different machines in a central location for monitoring and review.

The following /etc/syslog-ng/syslog-ng.conf will work with syslog-ng 3.3 or later, take all your local syslog messages and forward them to your technominds application via TCP:

version:3.3
source all_syslog {
  system();
};

destination technominds {
  syslog("technominds-receiver-syslog.TMCL.com"
    transport("tcp")
    port(514)
    template("@cee: $(format-json --pair message=\"$MSG\" --pair technominds-app-token=\"e8595c21-e379-4977-ad11-32d4210933ac\")\n")
  );
};

log { source(all_syslog); destination(technominds); };

Syslog-ng can also tail files and send logs from them to technominds. For each file, add a snippet like the one below to /etc/syslog-ng/syslog-ng.conf:

source messages {
  file("/var/log/messages"
    flags(no-parse) multi-line-mode(indented)
  );
};
log { source(all_syslog); destination(technominds); };

syslogd

Syslogd provides two system utilities which provide support for system logging and kernel message trapping. Support of both internet and unix domain sockets enables this utility package to support both local and remote logging.

Once your IP is registered, you can forward all your syslog messages to technominds via UDP. Just add the following line to your /etc/syslog.conf, then restart syslogd:

*.* @technominds-receiver-syslog.TMCL.com

Fluentd

Fluentd is an open source data collector, which lets you unify the data collection and consumption for a better use and understanding of data.

# Receive events from 24224/tcp
# This is used by log forwarding and the fluent-cat command
<source>
  @type forward
  port 24224
</source>

  @type http
  port 9880
</source>

  @type file
  path /var/log/fluent/access
</match>

    Setup

    To begin, download log4j from the official site. Extract the archive and copy the log4j-*.jar files in your project.

    Configure

    Set the log4j.xml file for your project. Its content is shown below:
    							
<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE log4j:configuration SYSTEM "log4j.dtd">
<log4j:configuration xmlns:log4j="http://jakarta.apache.org/log4j/">
<appender name="console" class="org.apache.log4j.ConsoleAppender">
<param name="Target" value="System.out" />
<layout class="org.apache.log4j.PatternLayout">
    <param name="ConversionPattern" value="%t - %m%n" />
</layout>
</appender>

<appender name="syslog" class="org.apache.log4j.net.SyslogAppender">
<param name="Facility" value="LOCAL0" />
<param name="SyslogHost" value="receiver-syslog.tmcl.com:514" />
<layout class="org.apache.log4j.PatternLayout">
    <param name="ConversionPattern"
									value="hfbab446-bsdb45-gjnjgisj4-sfjlsdj43" />
</layout>
</appender>

<root>
<priority value="debug" />
<appender-ref ref="console" />
<appender-ref ref="syslog" />
</root>
</log4j:configuration>
