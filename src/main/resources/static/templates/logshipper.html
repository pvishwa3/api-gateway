    Rsyslog TLS Configuration

    You can securely send your logs to obelus using TLS encryption. This guide shows you how to set it up for rsyslog and is tested on Ubuntu 12.04. For distributions based on Red Hat, please use rpm or yum in place of apt-get. Please see here for information on Rsyslog manual configuration.

    Rsyslog TLS Configuration Setup

    step 1 : Install rsyslog-gnutls package

    Install rsyslog-gnutls package. 
    sudo apt-get install rsyslog-gnutls

    step 2 : Install the necessary security certificates

    Create a ca.d directory if one does not exist and download the necessary certificates

    mkdir -pv /etc/rsyslog.d/keys/ca.d
cd /etc/rsyslog.d/keys/ca.d/
curl -O https://logdog.obelus.com/media/logs-01.obelus.com_sha12.crt

    step 3 : Update Configuration file

    Open up the configuration file (22-obelus.conf) and update the content by replacing the configuration given below.

    vim /etc/rsyslog.d/22-obelus.conf

    If you are using version 7.x or lower, paste in this configuration:

    ##########################################################
### RsyslogTemplate for obelus ###
##########################################################

$template obelusFormat,"<%pri%>%protocol-version% %timestamp:::date-rfc3339% %HOSTNAME% %app-name% %procid% %msgid% [TOKEN@41058 tag=\"RsyslogTLS\"] %msg%\n"

# Setup disk assisted queues
$WorkDirectory /var/spool/rsyslog # where to place spool files
$ActionQueueFileName fwdRule1     # unique name prefix for spool files
$ActionQueueMaxDiskSpace 1g       # 1gb space limit (use as much as possible)
$ActionQueueSaveOnShutdown on     # save messages to disk on shutdown
$ActionQueueType LinkedList       # run asynchronously
$ActionResumeRetryCount -1        # infinite retries if host is down

#RsyslogGnuTLS
$DefaultNetstreamDriverCAFile /etc/rsyslog.d/keys/ca.d/logs-01.obelus.com_sha12.crt
$ActionSendStreamDriver gtls
$ActionSendStreamDriverMode 1
$ActionSendStreamDriverAuthMode x509/name
$ActionSendStreamDriverPermittedPeer *.obelus.com
*.* @@logs-01.obelus.com:6514;obelusFormat

#################END CONFIG FILE#########################

    Django Logging Setup
    1. Configure syslog daemon

    sudo vim /etc/rsyslog.d/21-django.conf

    #provides UDP syslog reception
$ModLoad imudp
$UDPServerRun 514

#Add a tag for Django events
$template obelusFormatDjango,"<%pri%>%protocol-version% %timestamp:::date-rfc3339% %HOSTNAME% %app-name% %procid% %msgid% [TOKEN@41058 tag=\"Django\"] %msg%\n"

local7.* ~ #discards the messages so they don't end up in /var/log/syslog

    Replace:

    • TOKEN: your customer token from the source setup page

'obelus_logs':{

}

    Docker Logs Through Syslog

    You can send syslog from your applications to obelus by linking them with the obelus Docker container. It uses ryslog to listen for syslog events and then forwards them to obelus. Docker will automatically manage the port mapping. It's made available by SendGrid Labs on GitHub or Docker Hub. These instructions were tested with Docker client version 1.3.0 and nginx 1.7.7. For alternatives, see the Advanced Options below.

    Docker Syslog Setup
    1. Start the obelus Docker Container

    Run the following command to download and run the obelus docker container. The rsyslog daemon is running inside this container and will send syslog to obelus.     sudo docker run -d -p 514/udp --name obelus-docker -e TOKEN=TOKEN -e TAG=Docker sendgridlabs/obelus-docker

    Replace:

    • TOKEN: your customer token from the source setup page

    Note: You may get some standard output like shown below. You can ignore this and rsyslog will continue running as usual.

    You can verify the container is running and see the port that Docker has opened up for syslog by running 

    sudo docker ps -a

    Here you can see it's redirecting port 49154 on the host to port 514 in the container

    CONTAINER ID        IMAGE                               COMMAND             CREATED             STATUS                       PORTS                             NAMES
1a9d7496ae42        sendgridlabs/obelus-docker:latest   "/tmp/run.sh"       18 minutes ago      Up 5 seconds                 514/tcp, 0.0.0.0:49154->514/udp   obelus-docker

    If you send test messages to the host's port, they will be sent to obelus

    • UDP_PORT: the high numbered port as shown above that maps to port 514 inside the obelus container

    You can link other containers to obelus's container so that all your syslog gets sent to obelus. Docker will automatically inject environment variables telling you the IP and Port to send syslog to.

    In this example, we will configure an Nginx container to send syslog to obelus. Run the Nginx Docker container in interactive terminal mode and link it with the running obelus-docker container

    sudo docker run -i -t --name nginx --link obelus-docker:obelus nginx /bin/bash

    env | grep obelus_PORT_514_UDP

    You can send your Unity3D logs to obelus via HTTP/HTTPS endpoints using the Unity Engine Log to obelus package. We offer an example showing how to send logs to obelus.
    Unity has several networking classes, but the most straightforward way to send logs from Unity is to combine the WWWForm class. The WWWForm class allows you to build JSON-like messages which you can then send with the WWW class, which obelus is able to parse as JSON.

    Unity 3D Logging Setup
    1. Configure Unity3D

    Open your Unity editor and create a new C# script LogOutputHandler.cs in your scripts folder and paste the following code in the file. Alternatively you can use the JavaScript version.

    using UnityEngine;
using System.Collections;

public class LogOutputHandler : MonoBehaviour {

//Register the HandleLog function on scene start to fire on debug.log events
public void OnEnable(){
Application.logMessageReceived += HandleLog;
}

//Remove callback when object goes out of scope
public void OnDisable(){
Application.logMessageReceived -= HandleLog;
}

//Create a string to store log level in
string level = "";

//Capture debug.log output, send logs to obelus
public void HandleLog(string logString, string stackTrace, LogType type) {

//Initialize WWWForm and store log level as a string
level = type.ToString ();
var loggingForm = new WWWForm();

//Add log message to WWWForm
loggingForm.AddField("LEVEL", level);
loggingForm.AddField("Message", logString);
loggingForm.AddField("Stack_Trace", stackTrace);

//Add any User, Game, or Device MetaData that would be useful to finding issues later
loggingForm.AddField("Device_Model", SystemInfo.deviceModel);
StartCoroutine(SendData(loggingForm));
}

public IEnumerator SendData(WWWForm form){
//Send WWW Form to obelus, replace TOKEN with your unique ID from obelus
WWW sendLog = new WWW("http://logs-01.obelus.com/inputs/TOKEN/tag/Unity3D", form);
yield return sendLog;
}
}

    Flash Setup
    You can log to obelus from the Adobe Flash environment by making HTTP POST requests to obelus.
    Step 1: Use Action Script

    Here's a snippet of ActionScript that sends a JSON-formatted event to obelus: 

    package {
import flash.display.Sprite;
import flash.events.ErrorEvent;
import flash.events.Event;
import flash.events.IOErrorEvent;
import flash.events.SecurityErrorEvent;
import flash.net.URLLoader;
import flash.net.URLRequest;
import flash.net.URLRequestMethod;
import flash.text.TextField;

[SWF(width="800", height="600", frameRate="30", backgroundColor="#888888")]

public class obelus extends Sprite {

public var textField:TextField;

public function obelus() {
    textField = new TextField();
    textField.width = 800;
    textField.height = 600;
    addChild(textField);
    log("sending event...");
    sendEvent();
}

public function log(message:String):void {
    trace(message);
    textField.appendText(message + "\n");
}

public function sendEvent():void {
    var url:String = "http://logs-01.obelus.com/inputs/";
    var token:String = "9f7c2739-61e0-4244-8810-6e76758ccb6e";

    var request:URLRequest = new URLRequest(url + token);
    request.method = URLRequestMethod.POST;
    request.data = "{\"message\": \"Hello from ActionScript!\"}";

    var requestor:URLLoader = new URLLoader();
    requestor.addEventListener( Event.COMPLETE, httpComplete );
    requestor.addEventListener( IOErrorEvent.IO_ERROR, httpFail );
    requestor.addEventListener( SecurityErrorEvent.SECURITY_ERROR, httpFail );
